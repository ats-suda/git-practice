## 学んだことの要点

- Git はファイルの変更履歴を何度もセーブできる道具、GitHub はそれをネット上に置いて共有する場所。
- 基本サイクルは `add → commit → push`。最新を取り込むのは `pull`。
- テキストファイルは差分やマージができて相性が良い。バイナリ（CADやOffice等）はバージョン保存はできるが差分は見えにくい。
- 公開前には個人情報が入っていないか確認する。
