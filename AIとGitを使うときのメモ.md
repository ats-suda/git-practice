# AI（Claude）と一緒に Git / GitHub を使うときのメモ

Gitそのものの使い方ではなく、「AIに手伝ってもらいながらファイルを作って、GitHubに上げる」ときに知っておくと迷わないことをまとめたメモ。同じようにAIを使う人の参考になれば。

---

## 1. AIが作ったファイルは「どこにできる」のか

AI（Coworkなど）にファイルを作ってもらうと、多くの場合それは**AIの作業用フォルダ**にいったん保存される。Windowsだと `C:\Users\ユーザー名\AppData\...` のような、普段は開かない奥まった場所になりがち。

つまり「作ってもらった＝自分のリポジトリに入った」ではない。**GitHubに載るのは自分のリポジトリ（例：`Desktop\git-practice`）フォルダの中にあるファイルだけ**なので、ひと手間必要になる。

### ファイルを自分のリポジトリに持ってくる方法

- **フォルダで表示ボタン** … ファイルのカードにある「📁 フォルダで表示」を押すとエクスプローラーが開く。そこからコピー（Ctrl+C）して、`git-practice` フォルダに貼り付け（Ctrl+V）。
- **フォルダを接続してAIに置いてもらう** … Coworkでは対象フォルダを「接続」すると、AIがそのフォルダに直接ファイルを書き込める。コピペの手間が省ける。
- そのあとは自分で `git add → commit → push`（下記参照）。

---

## 2. Claude Cowork と Claude Code の違い

同じClaudeでも「動く場所」が違うため、できることが変わる。特に **GitHubへのpushを自動でやれるか** が実用上の分かれ目。

| | Claude Cowork | Claude Code |
|---|---|---|
| どこで動く | Claudeデスクトップアプリの中（隔離された作業環境） | 自分のPCの**本物のターミナル**の中 |
| 主な使い手 | 非エンジニアも。資料作り・ファイル整理・自動化 | 開発者。コーディング作業の委任 |
| ファイル | フォルダを「接続」して読み書き | 手元のファイルを直接読み書き |
| Gitのpush | **できない**（認証情報を持たない／ターミナルに入力できない） | **できる**（自分のGit設定・GitHubログインをそのまま使える） |
| 向いていること | 文章・資料・調べもの・下ごしらえ | コード編集、テスト実行、commit〜pushまで一気通貫 |

### なぜ Cowork は push まで代われないのか

1. Coworkのシェルは**隔離された環境**で、自分のGitHubログイン情報（トークン等）が入っていない。だから `git push` は認証で弾かれる。
2. 安全のため、Coworkは自分のターミナル（Git Bash など）に**文字を打ち込めない**ようになっている。

そのため Cowork の役割は「ファイルを作る・整える・リポジトリフォルダに置く」まで。最後の `push` は自分の手（または GitHub Desktop）で行う。

### 「pushまで全部自動」にしたいなら Claude Code

Claude Code はターミナル上で動き、自分の認証情報を使えるので、「この2ファイルをコミットしてpushして」と頼むだけで最後まで自動でやってくれる。定型的なGit作業を任せたいならこちらが速い。

---

## 3. 最後の push（自分でやる部分）

ファイルがリポジトリフォルダに入ったら、Git Bash で:

```bash
cd ~/Desktop/git-practice
git status      # 追加したファイルが Untracked で出るか確認
git add .
git commit -m "○○を追加"
git push
```

コマンドが苦手なら GitHub Desktop で「Summary記入 → Commit → Push」でも同じ。

---

## 4. AIとGitを使うときの注意点

- **push前に個人情報チェック** … AIが作った文章に実名・メール・秘密情報が紛れていないか確認してから公開する。`grep -rn "探したい文字" .` で検索できる。Publicリポジトリは世界に公開される。
- **認証情報はAIに渡さない** … パスワードやGitHubトークンをチャットに貼らない。pushは自分の環境で行う。
- **どこにファイルができたか必ず確認** … 「作った」と「自分のリポジトリに入った」は別。場所を確かめるクセをつける。
- **AIは差分の説明が得意** … 「このコミットで何が変わった？」「この英語のエラーは何？」と聞くと、色分けやメッセージの意味を解説してくれる。詰まったらそのまま貼って聞くのが早い。
- **コミットメッセージも頼める** … 変更内容を伝えれば「memoに2行目を追加」のような分かりやすいメッセージを提案してくれる。

---

## 5. ざっくり使い分け

- **文章・資料・調べもの・下ごしらえ** → Cowork が得意。
- **コードを書く・テストする・commitからpushまで一気に** → Claude Code が得意。
- **共通の心構え** → 「AIが作る」→「自分のリポジトリに置く」→「（自分の権限で）push」→「公開前に中身を確認」。この流れを押さえておけば、どちらを使っても迷わない。
